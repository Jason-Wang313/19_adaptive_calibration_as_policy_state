# Full-Scale Execution Plan For Paper 19

## Current Claim

Paper 19 argues that calibration drift should be represented as policy state.
The current v2 manuscript treats Calibration-State Control (CSC) as a recurrent
control interface: the policy conditions on an online estimate of the current
action-observation calibration map instead of treating calibration as fixed
offline preprocessing or generic robustness noise.

The existing evidence is a deterministic 2D hidden-calibration drift simulator:
14,400 main rollouts plus a 2,400-rollout Windowed SysID hostile baseline.
Current headline results show CSC nearly matching oracle under moving drift and
beating frozen-start calibration in abrupt and severe drift modes. The existing
Downloads PDF is only 6 pages, so it is not final under the full-scale batch
standard.

## Current Gaps

- Manuscript length and evidence depth are far below the 25-page final-paper
  threshold.
- Evidence is still concentrated in four drift modes and a small controller
  set.
- The strongest claim is the policy-state interface, but v2 only partially
  separates the interface from a specific RLS estimator.
- Observability and excitation are named as limitations but not stressed
  experimentally.
- Ill-conditioning, latency, outliers, action saturation, and sensor noise are
  not mapped.
- Planning/control impact is measured by success and final error, but not by
  regret, adaptation lag, invalid/unsafe command burden, or oracle gap.
- Negative controls are too weak: the paper needs shuffled/stale/scalar
  calibration-state controls to rule out the idea that any recurrent context
  suffices.
- Docs still describe v2 as workshop-only / strong-revise and must be upgraded
  only if the evidence actually supports a narrower final claim.

## Full-Scale Experiment Families

All families must be generated by a RAM-light script that streams rows to
`results/full_scale/`, writes progress after each family, and never requires
holding full rollout tensors in memory.

### Family A: Main Controller Comparison

Goal: reproduce and extend the v2 core comparison.

Controllers:
- nominal offline controller;
- robust low-gain controller;
- frozen-start calibration controller;
- residual-bias adapter;
- windowed SysID;
- CSC/RLS policy-state controller;
- oracle calibration controller.

Drift modes:
- static;
- slow random walk;
- abrupt bumps;
- severe random walk;
- sinusoidal drift;
- scale-only drift;
- rotation-only drift;
- shear-heavy drift.

Metrics:
- success rate;
- final error;
- tail tracking error;
- path efficiency;
- total command norm;
- calibration-estimation error;
- oracle gap;
- adaptation lag after drift events.

### Family B: Estimator And Interface Ablations

Goal: separate the policy-state interface from the exact estimator.

Variants:
- CSC with RLS forgetting sweep;
- CSC with different prior scales/ridge values;
- windowed SysID with window sweep;
- no-forgetting estimator;
- calibration estimate computed but not fed to action computation;
- scalar drift-norm state only;
- stale initial estimate;
- shuffled calibration estimate from another episode.

Key question: does the advantage come from carrying the right action-observation
map inside the policy loop, or merely from extra memory/context?

### Family C: Observability And Excitation Stress

Goal: test the explicit boundary that CSC helps only when calibration is locally
observable from recent transitions.

Stress axes:
- goal diversity and goal period;
- action-command clipping;
- low-excitation trajectories;
- small exploratory dither;
- update gating for tiny commands;
- repeated goals that make the calibration map poorly identifiable.

Metrics:
- success and final error;
- estimator condition number;
- calibration error;
- rank/excitation score of recent commands;
- failures attributable to insufficient excitation.

### Family D: Noise, Latency, Dropout, And Outliers

Goal: map robustness under realistic sensing/control imperfections.

Stress axes:
- observation noise sweep;
- action execution noise sweep;
- delayed observation update;
- dropped residual updates;
- outlier observations;
- mixed noise plus abrupt drift.

Metrics:
- success degradation curve;
- oracle gap;
- calibration error under noise;
- outlier recovery time;
- command overshoot.

### Family E: Ill-Conditioning And Safety-Oriented Control

Goal: test the failure mode where the calibration map becomes nearly singular or
the inverse controller amplifies noise.

Stress axes:
- minimum scale values;
- anisotropy ratio;
- shear magnitude;
- inverse ridge/clipping sweep;
- condition-aware abstention or low-gain fallback.

Metrics:
- condition number;
- command norm;
- final error;
- fallback/abstention rate;
- success under near-singular maps.

### Family F: Drift Event Recovery

Goal: quantify how quickly controllers recover after discrete calibration
events.

Stress axes:
- one abrupt event;
- repeated bumps;
- late bump after convergence;
- alternating rotation and scale events;
- varying bump magnitude.

Metrics:
- steps to recover below error threshold;
- peak post-event error;
- adaptation lag;
- cumulative regret versus oracle.

### Family G: Planning/Control Utility

Goal: show that calibration state matters to action choice, not only parameter
estimation.

Scenarios:
- multi-goal reaching under drift;
- goal switching with time budget;
- action selected by nominal, frozen, windowed, CSC, and oracle predicted maps;
- local action regret measured against the oracle action.

Metrics:
- action regret;
- goal completion rate;
- final/tail error;
- command burden;
- oracle gap.

### Family H: Negative Controls

Goal: rule out weaker explanations.

Controls:
- random calibration state;
- shuffled calibration state across episodes;
- stale-but-privileged initial calibration;
- scalar recent residual norm only;
- windowed context without matrix structure;
- oracle delayed by several steps.

Expected finding: generic context can help somewhat, but structured current
calibration state is required to close the oracle gap under moving drift.

## Writing Expansion Strategy

The final manuscript must reach at least 25 pages through real content:
- expanded introduction and claim boundary;
- related-work separation from online calibration, visual servoing, system ID,
  and latent-context adaptation;
- formal ambiguity example;
- CSC method and estimator-independent policy-state interface;
- full experiment design with eight families;
- results with positive findings and negative results;
- limitations and falsifiers;
- appendix with family protocols, metrics, calibration observability, failure
  taxonomy, reviewer self-attacks, reproducibility checklist, and final audit.

Do not pad. If the PDF is short, add missing experimental protocols,
mathematical/algorithmic details, failure analysis, or reproducibility content.

## RAM-Light Execution Strategy

- Run all families sequentially.
- Stream seed/block rows to CSV as each family completes.
- Store only compact aggregates needed for figures/tables.
- Use deterministic seeds and small NumPy arrays.
- Avoid large rollout tensor caches.
- Write `results/full_scale/progress.json` after every family.
- Keep generated plots compact PDF/PNG files.
